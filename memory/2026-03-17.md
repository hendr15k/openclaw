- **Schema:** 7 tables (object, tree, content, metadata, trash, relation, journal)
- **Object Types:** 1=root, 2=collections, 3=folders, 4=documents, 5=pages

### DB Schema Learnings
- **Hierarchy:** root(1) → collections(2) → folders(3) → documents(4) → pages(5)
- **Tree table:** `parent_id`→`child_id` links folder→doc (type 3→4), doc→doc (4→4)
- **Pages NOT in tree** — they're linked via content table object_id
- **Metadata:** `key`/`value` pairs; `name` for folders, `title` for documents
- **Content table:** `signature` = SHA256 hash, `path` = file extension (.png/.pdf), `page_id` = internal page type
- **dat/ folder:** UUID-based file storage; first 3 hex chars as subdirectory (e.g., `dat/401/4013b9e6-...png`)
- Pages (type 5) have **no metadata entries** at all
- Documents show `child_page_count = 0` (pages tracked elsewhere)
- **SQL gotcha:** SQLite requires single quotes for string literals (double quotes = column names)
